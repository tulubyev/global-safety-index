# Global Safety Index — Методология расчёта

## Формула Safety Score

```
SafetyScore = normalize( w₁ × ConflictScore + w₂ × DisasterScore + w₃ × FoodScore )
```

- Веса `w₁, w₂, w₃` задаются пользователем через слайдеры и нормализуются на сервере до суммы = 1
- Итоговая оценка применяет **sqrt-нормализацию** глобально по всем 235 странам:
  `score = √((raw − min) / (max − min)) × 100`
- Это растягивает середину диапазона и компенсирует скошенность (несколько зон конфликта с очень высокими значениями)
- **0 = самая безопасная страна в наборе данных, 100 = самая опасная**

## Источники данных

### 1. ConflictScore (риск вооружённых конфликтов и терроризма)

Берётся **максимум** из двух независимых источников:

#### UCDP GED 23.1 — Uppsala Conflict Data Program
- **Что измеряет:** задокументированные боевые потери в вооружённых конфликтах
- **Охват:** 91 страна, 1989–2023
- **Метод:** суммируются жертвы начиная с 2015 года, min-max нормализация → 0–100
- **Ограничение:** учитывает только конфликты на территории страны; агрессоры (Россия в войне на Украине) не получают баллов
- **URL:** https://ucdp.uu.se/downloads/ged/ged231-csv.zip *(публичный, без токена)*
- **Файл-парсер:** `backend/src/parsers/ucdpParser.js`

#### Our World in Data / GTD — Global Terrorism Database
- **Что измеряет:** число смертей от терактов по годам
- **Охват:** 187 стран, 1970–2021 (через OWID-агрегацию данных START UMD)
- **Метод:** взвешенная сумма за 2010–2021 с экспоненциальным затуханием (half-life = 5 лет), логарифмирование, min-max нормализация → 0–100
- **Преимущество перед UCDP:** включает страны спонсоры/жертвы терроризма (Иран, Нигерия, Франция, Норвегия)
- **URL:** https://ourworldindata.org/grapher/terrorism-deaths.csv *(публичный, без токена)*
- **Файл-парсер:** `backend/src/parsers/gtdParser.js`

**Итоговый ConflictScore** = `GREATEST(ucdp_score, gtd_score)`

---

### 2. DisasterScore (риск стихийных бедствий)

#### World Bank — EN.CLC.MDAT.ZS
- **Что измеряет:** доля населения, пострадавшего от стихийных бедствий (среднее 1990–2009)
- **Охват:** 156 стран
- **Метод:** min-max нормализация исходного % → 0–100
- **Ограничение:** данные устарели (1990–2009); для стран без данных используется mock ≈ 25–30
- **API:** `https://api.worldbank.org/v2/country/all/indicator/EN.CLC.MDAT.ZS?format=json&mrv=1&per_page=300`
- **Файл-парсер:** `backend/src/scripts/importDisasterRisk.js`

---

### 3. FoodScore (продовольственная безопасность)

#### World Bank — SN.ITK.DEFC.ZS
- **Что измеряет:** распространённость недоедания (% населения)
- **Охват:** 163 страны, последний доступный год (~2021–2022)
- **Метод:** min-max нормализация % недоедания → 0–100
- **Логика:** выше % недоедания → выше FoodScore → менее безопасная страна
- **Страны с < 2.5% недоедания** (большинство развитых стран) получают FoodScore ≈ 0
- **API:** `https://api.worldbank.org/v2/country/all/indicator/SN.ITK.DEFC.ZS?format=json&mrv=1&per_page=300`
- **Файл-парсер:** `backend/src/parsers/worldBankParser.js`

### 4. DisasterScore — обновлён (INFORM Risk Index 2026)

#### INFORM Risk Index — EU Joint Research Centre (JRC)
- **Что измеряет:** комплексный индекс природных рисков: землетрясения, наводнения, цунами, тропические циклоны, засухи, эпидемии
- **Охват:** 191 страна, ежегодное обновление
- **Метод:** sub-score "Natural" (0–10) масштабируется до 0–100
- **Заменяет:** устаревшие данные World Bank 1990–2009
- **URL:** `https://drmkc.jrc.ec.europa.eu/inform-index/Portals/0/InfoRM/2026/INFORM_Risk_2026_v072.xlsx` *(публичный, без токена)*
- **Файл-парсер:** `backend/src/parsers/informParser.js`

---

### 5. SeismicScore (сейсмическая активность — реалтайм)

#### USGS Earthquake Hazards Program
- **Что измеряет:** энергия землетрясений M4.5+ за последние 30 дней
- **Охват:** ~33 страны (по данным последнего месяца), глобальный охват
- **Метод:** сумма сейсмической энергии `10^(1.5×mag)` по стране, log-нормализация → 0–100
- **Обновление:** реалтайм (каждые ~5 минут на USGS)
- **URL:** `https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/4.5_month.geojson` *(публичный, без токена)*
- **Файл-парсер:** `backend/src/parsers/usgsParser.js`
- **Ограничение:** country определяется из текстового поля `place`; ~75 уникальных мест → 33 страны

## Известные ограничения

1. **Газа / Палестина** — conflict=0: война октября 2023 не попала в GTD 2021 и UCDP 23.1
2. **Россия** — ConflictScore только по GTD (теракты Северного Кавказа); война на Украине отражена в данных **Украины**, не России — так работает UCDP
3. **Йемен** — данные только GTD (7181 смертей 2010-2021); UCDP 23.1 не матчит по коду
4. **GTD до 2021** — атаки 2022–2024 (Украина, Газа, Сахель) не отражены
5. **Disaster данные 1990–2009** — не учитывают недавние катастрофы (Турция 2023, Марокко 2023)
6. **~50 стран** используют mock-данные по disaster/food из-за отсутствия в World Bank

## Актуализация данных

### Автоматическая (cron, backend)
- GDELT конфликты: каждые 15 минут (но отключён из-за медиа-смещения)

### Ручная
```bash
# Обновить данные о терроризме (GTD через OWID — обновляется ~ежегодно)
node backend/src/scripts/importGtd.js

# Обновить продовольственные данные (World Bank — ежегодно)
node backend/src/scripts/importWorldBank.js

# Обновить данные о стихийных бедствиях
node backend/src/scripts/importDisasterRisk.js

# Обновить UCDP (новый релиз: UCDP GED 24.1 ожидается ~июнь 2024)
# Изменить URL в backend/src/parsers/ucdpParser.js на новую версию
node backend/src/scripts/importUcdp.js
```

### Для подключения ACLED (реалтайм конфликты)
1. Зарегистрироваться на https://developer.acleddata.com/
2. Добавить `ACLED_KEY=...` в `backend/.env`
3. Реализовать `backend/src/parsers/acledParser.js`
